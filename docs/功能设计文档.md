# 功能设计文档

## 1. 项目概述

### 1.1 目标
开发一个桌面端管理软件，以**多标签**形式集中管理 **Claude Code** 与 **Gemini CLI** 的会话。用户可在不同标签中并行操作多个 AI 编码助手会话，并对历史会话进行浏览与恢复。

### 1.2 核心定位
- 不是聊天客户端，而是**多 CLI 终端管理器 + 会话索引器**。
- 每个标签 = 一个真实的 CLI 进程（PTY 托管），保留官方 100% 的交互能力（斜杠命令、权限提示、MCP 等）。
- 应用层负责：标签编排、工作目录管理、历史会话浏览与恢复、设置。

### 1.3 技术路线
**PTY 终端托管**：Rust 用 `portable-pty` 启动真实 `claude` / `gemini` 进程，前端用 xterm.js 渲染终端画面，Tauri Channel 双向传输字节流。

> 对话内容由 CLI 自身存储于 `~/.claude/projects/**/*.jsonl`，应用不重复存储，仅做索引、浏览与恢复。

---

## 2. 技术选型

| 层 | 选型 | 说明 |
|----|------|------|
| 桌面壳 | Tauri 2.x | 体积小、多窗口、IPC 强 |
| 前端框架 | React + TypeScript + Vite | 生态大、终端/Markdown 轮子多 |
| 状态管理 | Zustand | 轻量，适合标签树状态 |
| 终端渲染 | xterm.js + addons | fit 自适应 / web-links / search / webgl |
| PTY | Rust `portable-pty` | Windows 走 ConPTY |
| 流式通道 | Tauri `Channel` | 高吞吐双向字节流 |
| 本地存储 | SQLite (`rusqlite`) | 流行稳定、单文件、零运维 |
| 异步运行时 | tokio | |

存储位置：Tauri `app_data_dir`（Windows `%APPDATA%`），SQLite 单文件。

---

## 3. 数据模型（SQLite）

应用只存元数据，**不存对话内容**。

```sql
-- 标签
tabs(
  id          INTEGER PRIMARY KEY,
  provider    TEXT,      -- 'claude' | 'gemini'
  cwd         TEXT,      -- 工作目录
  session_id  TEXT,      -- 绑定的官方会话 id（可空）
  title       TEXT,
  sort_order  INTEGER,
  is_active   INTEGER,
  created_at  INTEGER,
  updated_at  INTEGER
);

-- 设置
settings(
  key   TEXT PRIMARY KEY,
  value TEXT
);
```

会话历史不入库，运行时解析官方 JSONL 文件获取。

## 4. 功能拆解（按期）

### 第一期 · MVP：单标签跑通 PTY
- **P1-1 项目脚手架**：Tauri 2 + React + TS + Vite 初始化，目录结构。
- **P1-2 Rust PTY 核心**：`portable-pty` 启动 `claude`（指定 cwd）；后台线程读 PTY 输出经 Channel 推前端；前端输入写回 PTY。
- **P1-3 前端终端**：xterm.js 挂载、fit 自适应、`onData` 发送输入、resize 同步 cols/rows。
- **P1-4 进程生命周期**：启动 / 退出 / 强制结束 / 崩溃状态反馈。

> 验收：单窗口内可正常交互的 Claude Code 终端。

### 第二期 · 多标签 + 持久化
- **P2-1 多标签管理**：增 / 删 / 切换，每标签独立 PTY，切换只显示对应终端。
- **P2-2 SQLite 接入**：`tabs` / `settings` 表，标签元数据持久化。
- **P2-3 重启恢复**：重开应用恢复标签布局，PTY 重新拉起（可选 `--resume` 续接）。
- **P2-4 新建标签 UI**：选工作目录 + 选 provider（先 Claude）+ 新会话/恢复会话选项。

### 第三期 · 会话管理（核心诉求）
- **P3-1 会话浏览器**：解析 `~/.claude/projects/**/*.jsonl`，列出历史会话（标题 / 时间 / 首条消息 / cwd）。
- **P3-2 恢复会话**：点击历史会话 → 新标签 `claude --resume <session-id>`。
- **P3-3 设置页**：claude 可执行路径、默认 cwd、主题、字体。
- **P3-4 打磨**：终端内搜索、复制、Gemini provider 抽象接入、用量统计、导出。

## 5. 关键技术风险

| 风险 | 说明 | 应对 |
|------|------|------|
| Windows ConPTY | 颜色 / 光标偶有异常 | xterm.js 兼容，必要时调参 |
| 高吞吐流式 | CLI 刷屏快易卡 UI | Channel 批量 flush / 背压 |
| 可执行路径与认证 | 需定位 `claude`，登录交给 CLI | 自动探测 + 手动配置，应用不介入认证 |
| PTY resize 同步 | cols/rows 不同步会换行错乱 | xterm resize 实时同步后端 |

## 6. Provider 抽象（面向扩展）

统一接口，新增 provider 不改 PTY 核心：

```rust
trait Provider {
    fn command(&self, cwd: &Path, resume: Option<&str>) -> Command; // 构造启动命令
    fn sessions_dir(&self) -> PathBuf;                              // 历史会话目录
    fn parse_session(&self, path: &Path) -> SessionMeta;           // 解析会话元数据
}
```

- `ClaudeProvider`：`claude` / `~/.claude/projects/` / JSONL 解析
- `GeminiProvider`：第三期接入
